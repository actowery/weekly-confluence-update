A Claude Code skill that drafts your weekly status update into a shared Confluence weekly-report page by researching Jira, Slack, and Outlook for the date range, shows you a local HTML preview of exactly what it plans to add, and only publishes to Confluence after you explicitly approve.

Designed for engineering managers / team leads filling out a shared weekly roll-up where many teams write into the same page.

## Prerequisites

Claude Code (or any client that can load skills) with these MCP servers connected:

| Tool | Used for | Required? |
|---|---|---|
| Atlassian MCP (Confluence + Jira) | Fetch/update the page; search Jira for team activity and releases | Yes |
| Slack MCP | Search public (and optionally private) messages; resolve user IDs | Yes |
| Outlook / M365 MCP | Search email for the date window | Recommended |
| `gh` CLI | Search GitHub for merged + in-flight PRs by team members | Optional (needed only if your team config sets `github.orgs`) |

Private Slack search is **off by default**. Enable it per-team by setting `slack_search_mode: "public_and_private"` in your team config (via the init flow), or decline per-run with `public only`.

If any tool is missing, the skill still runs but the research phase is thinner — it'll tell you which sources it couldn't reach.

## Quick start

Copy the skill into your `~/.claude/skills/` directory (or wherever your Claude client loads skills from).

### 1. Set up your team once

Tell Claude:
```
init weekly-confluence-update for <TeamName>
```

Claude will:
- Ask you for team member names (paste them in any form — comma-separated, newline-separated, "add X, Y, Z")
- Resolve each to Atlassian + Slack IDs via MCP lookups
- Ask you (optionally) for Jira project keys, Slack channels, and email keywords associated with your team's work
- Save everything to `${XDG_CONFIG_HOME:-$HOME/.config}/weekly-confluence-update/teams/<team-slug>.json` — outside the skill install dir so upgrades don't wipe your roster

You can re-run init anytime to add/remove members.

### 2. Run the weekly update

```
<confluence-page-url> <TeamName>
```

Claude will:
1. Fetch the page and figure out which rows you're @-mentioned in
2. Parse the date range from the page title
3. Search Jira / Slack / Outlook for your team's work in that window
4. Draft paragraph-per-topic content in the house style
5. Open a local HTML preview in your browser with additions highlighted yellow
6. Wait for you to reply `approve`, `edit <section>: <change>`, or `cancel`

It only calls `updateConfluencePage` after you explicitly approve.

## If your page template is different

The skill looks for 2-column `Category | Update` tables with @-mentioned prompt paragraphs containing `"provide weekly updates"` and rows labeled "Challenges", "Product Releases Completed", etc. If your Confluence template uses different wording (e.g. "Blockers" instead of "Challenges", "share updates on" instead of "provide weekly updates"), override the defaults in your team config:

```json
"page_layout": {
  "team_only_row_labels": ["blockers", "shipped", "team news"],
  "prompt_phrase_regex": "share your updates|please update"
}
```

## Safety rules the skill follows

- **Append-only.** Never deletes or modifies content other people authored.
- **Team-scoped.** Releases/challenges entries only cover your team, not the whole page.
- **No fabrication.** Every bullet must trace to a real Jira ticket, Slack message, or email. If research is thin for a section, the preview says so rather than filling with vague text.
- **Approval-gated.** Nothing publishes without an explicit `approve` from you.

## Running the smoke test

After editing `parse_page.py` or `render_preview.py`, run:
```
python3 tests/smoke_test.py
```
It exercises every subcommand (section mapping, date extraction, patch building, sentinel stripping, HTML rendering) against sanitized fixtures in `tests/fixtures/`. Exits non-zero on any failure. No network calls.

To regenerate the fixtures (e.g. if you've changed the ADF shape the skill expects):
```
python3 tests/make_fixtures.py
```
The generator is hand-written — open it to see what a minimal valid weekly-report page looks like.

## Troubleshooting

| Symptom | Likely cause |
|---|---|
| "No sections found" | Page template doesn't match defaults — set `page_layout` overrides in team config |
| Roster discovery returns wrong people | Re-run `init` and give the correct names directly |
| Slack search empty | Team uses private channels only; skill defaults to public. Authorize private channel search for the run if needed |
| Preview highlights look off | Look at `modified.json` in the workspace — the `_skillAdded: true` sentinel is what highlighting keys on |
